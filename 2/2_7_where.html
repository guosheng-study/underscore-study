2.7 - where(特性查找)

语法: _.where(list, properties)

关键词：按特性查找。

筛选集合中符合指定条件的元素并返回新的结果集，list为筛选的目标集合,properties键值对(指定条件)，如{sex:'Male'},若为多个，则以逗号分隔， 如{sex:'Male',age,10}

a) - 从list中筛选出有properties特性的元素
var list = [{ 'name': 'guofucheng', 'sex': 'Male' }, { 'name': 'fanbingbing', 'sex': 'Female' }, { 'name': 'liudehua', 'sex': 'Male' }]; var properties = {'sex' : 'Male'}; //筛选出list集合中sex为Male的元素 var stars = _.where(list, properties); console.log(stars); //=> [Object {name="guofucheng", sex="Male"}, Object {name="liudehua", sex="Male"}]
http://jsfiddle.net/kyo4311/6jvgU/

_.where()方法第一个参数传的是list(集合);第二个参数传的是一个特性(比如说是一个对象)，不是一个回调函数。返回的数组中的元素都拥有{'sex' : 'Male'}特性。事实上如果不用_.where()，你们也可以用_.filter()来实现他。

b) - 用filter实现同样的功能
var list = [{ 'name': 'guofucheng', 'sex': 'Male' }, { 'name': 'fanbingbing', 'sex': 'Female' }, { 'name': 'liudehua', 'sex': 'Male' }]; var properties = {'sex' : 'Male'}; var stars = _.filter(list, function(o){ return o.sex === 'Male'; }); console.log(stars); //=>[Object {name="guofucheng", sex="Male"}, Object {name="liudehua", sex="Male"}]
http://jsfiddle.net/kyo4311/qq2Yn/

_.filter()和_.where()一起对比来理解一下的话，_.where()是直接传入了一个特性，而_.filter()的特性在回调函数里返回。如果需要过滤不同的特性的时候可以考虑用_.filter()，如果list经常变，那特性不变的情况，可以考虑使用_.where()。

c) - 设置多一点特性
var list = [{ name: 'guofucheng', sex: 'Male', age : 18 }, { name: 'fanbingbing', sex: 'Female', age : 18 }, { name: 'liudehua', sex: 'Male', age : 17 }]; //可以设置多一点条件 var properties = { sex: 'Male', age : 18 }; var stars = _.where(list, properties); console.log(stars); //=>[Object {name="guofucheng", sex="Male", age=18}]
http://jsfiddle.net/kyo4311/cq8Vw/

当数组里的元素是一个对象并需要复筛选元素的时候，比较适合用_.where()。
